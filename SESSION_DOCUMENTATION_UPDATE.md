# Session Documentation Update

Purpose: capture current implementation and documentation state while work is still active.

Use this workflow when:

- repo structure changed
- content workflow changed
- planning docs need refresh
- implementation may continue later

Do not use this for final closeout.

## Documentation Routing

- Stable docs: `documentation/...`
- Change records: `documentation/changes/...`

## Workflow

1. Audit current changed files and relevant verification state.
2. Update planning docs when current execution status changed.
3. Add or update a change doc for meaningful implemented work.
4. Update runbooks or architecture docs if the operating model changed.
5. Report what changed and any follow-up items.

## Notes

- Keep this file stable; continuity should come from change docs.
