# Passage Page Scripture-Context Standardization

Date: 2026-06-29
Scope: documentation

## Summary

- Updated `documentation/architecture/CONTENT_EXECUTION_STANDARD.md`.
- Standardized passage pages so the top `Passage` section holds the full default translation text.
- Defined `Context` as Scripture context rather than process or navigation context.
- Defined `Explanation` as direct interpretive prose drawn from source teaching content without public meta references to "the study".
- Locked in that supporting passage links should open in a new tab.

## Why

- Passage pages need to help readers interpret Scripture in context rather than function as thin navigation shells.
- Public prose should teach directly instead of describing internal content sources or generation workflow.
- The pattern established for `1 John 5:14-15` needs to be durable so future passage pages are built the same way by default.

## Notes

- This change establishes a repo-level rule, not a one-off content preference.
- Future passage-page work should follow this standard unless the user explicitly requests a different structure.
