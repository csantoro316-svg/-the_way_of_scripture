# Passage Library Foundation Plan

Status: Active
Started: 2026-07-05

## Objective

Build the passage library into a canonical, scalable system for full-Bible coverage, with stable source structure, restrained content standards, reliable validation, and a book-by-book expansion workflow.

## Why This Plan Exists

The current passage library began as a study-support layer built around referenced passages. That was sufficient for generated study links, but it is not the right long-term model for a full Bible library. The project now needs:

- canonical section-based source files
- stable browse/navigation behavior
- clear content-layer distinctions
- validation rules before volume increases
- a repeatable workflow for adding books

## Target Model

### Canonical navigation

- Book
- Chapter
- Section
- Verse

### Canonical content unit

The canonical page and source unit is a section within a chapter, not an isolated verse and not a whole chapter exposition page.

### Passage page structure

- Context
- Common Interpretations
- Current Position
- Deeper Study

### Editorial intent

- `Context` stays close to the passage, its local flow, and its historical or literary setting.
- `Common Interpretations` acknowledges major readings without forcing a final doctrinal commitment on every passage page.
- `Deeper Study` points readers to fuller site arguments only where those arguments have actually been developed.

## Phases

## Phase 1: Foundation

Goal: lock the structure before scaling content.

Deliverables:

- final section-based browse/navigation pattern
- canonical passage source schema with section title support
- passage page structure aligned to `Context`, `Common Interpretations`, and `Deeper Study`
- dedicated validation tooling for passage data and generator integrity
- proof-of-concept short book completed end-to-end

Notes:

- `Philemon` is the current proof of concept.
- Validation should distinguish between structural errors and editorial warnings so the repo can improve incrementally without blocking on every incomplete legacy file.

## Phase 2: Schema Consolidation

Goal: move the passage source layer toward a fully normalized standard.

Deliverables:

- documented section JSON schema
- standardized fields for:
  - reference metadata
  - section title
  - translation blocks
  - context lines
  - common interpretation lines
  - current position lines
  - deeper study links
- reduced reliance on fallback generated commentary

## Content Standards For Passage Pages

- Do not present autogenerated fallback prose as if it were a reviewed doctrinal conclusion.
- Prefer restrained summary over confident interpretation when a passage has not been formally studied.
- Use `Current Position` only where the site is prepared to state its present conclusion plainly; otherwise say so directly and leave the matter open.
- Use `Deeper Study` only when the site has a developed treatment worth standing behind.
- Preserve room for more than one common reading where the passage genuinely bears interpretive tension.

## Validation Strategy

Validation is a required part of the workflow.

### Structural checks

- valid JSON
- required reference fields
- consistent label and slug
- valid citation format
- sortable chapter and verse ranges
- duplicate reference detection
- overlap detection within the same chapter

### Editorial checks

- missing section title
- missing BSB text
- missing context
- missing common interpretation content
- broken deeper-study references

Structural failures should block progress. Editorial gaps should surface as warnings until the corpus is normalized.

## Working Rules

- Source first, generated output second.
- Do not patch generated browse/library output as the primary source of truth.
- Expand books only after validation passes.
- Keep navigation work and interpretation work separate whenever possible.

## Immediate Next Steps

2. Run validation against the current passage source corpus.
3. Review and fix any structural failures.
4. Continue Phase 1 hardening against the active proof of concept.
5. Move into book-by-book completion for study-linked books once the foundation is stable.
